# PDD Workflow Transaction Templates

These templates define the JSON-LD shape the Flutter workflow shell should send
to Fluree after the generated shape2flutter forms produce payload data.

The templates are not a second data model. They use canonical NIAS predicates
and the document schema IRIs already defined in the SHACL assets.

## Common Document Envelope

Every submitted workflow document should include:

| Field | Requirement |
| --- | --- |
| `@id` | Stable document IRI minted by the application. |
| `@type` | Always includes `data:Document`; may include workflow-specific classes. |
| `nias-o:resourceIpfsUri` | IPFS/IPNS URI for the persisted document payload. |
| `nias-o:documentSchema` | Canonical NIAS document schema IRI. |
| `nias-o:isEncrypted` | Boolean encryption flag. |
| `nias-o:documentAuthor` | Author or reviewer IRI. |
| `nias-o:authProof` | Canonical proof concept such as `nias-cs:eddsa-signature`. |
| `nias-o:hasWorkflowSubmission` | One embedded or referenced workflow submission node. |

Every workflow submission should include:

| Field | Requirement |
| --- | --- |
| `@type` | `nias-o:WorkflowDocumentSubmission`. |
| `nias-o:submittedDocument` | The submitted `data:Document`. |
| `nias-o:workflow` | PDD workflow IRI. |
| `nias-o:workflowStep` | Step resource with `rdfs:label`. |
| `nias-o:workflowSubject` | Project IRI. |
| `nias-o:workflowDocumentSubmittedBy` | Submitting platform user. |
| `nias-o:workflowDocumentRecipient` | Registry or assigned recipient. |
| `nias-o:workflowSubmissionConsensusMessage` | Hedera topic message evidence. |

## PDD Section A Submission

Document schema:
`https://nova.org.za/novaimpactaccountingstandard/document-schema/PDDxA-1.0.0`

Expected SHACL coverage:

- `nias-o:DocumentShape`
- `nias-o:PddSectionAReportShape`
- `nias-o:ProjectDesignShape`

```json
{
  "@context": "dataRequirements/fluree/context.jsonld",
  "@id": "nias:documents/{project-slug}/pdd-section-a/{submission-id}",
  "@type": ["data:Document"],
  "nias-o:resourceIpfsUri": {
    "@type": "xsd:anyURI",
    "@value": "ipfs://..."
  },
  "nias-o:documentSchema": {
    "@id": "nias:document-schema/PDDxA-1.0.0"
  },
  "nias-o:isEncrypted": false,
  "nias-o:documentAuthor": {
    "@id": "nias:users/{developer-id}"
  },
  "nias-o:authProof": {
    "@id": "nias-cs:eddsa-signature"
  },
  "nias-o:reportContent": {
    "@id": "nias:reports/{project-slug}/pdd-section-a/{submission-id}"
  },
  "nias-o:hasWorkflowSubmission": {
    "@id": "nias:submissions/{project-slug}/pdd-section-a/{submission-id}",
    "@type": "nias-o:WorkflowDocumentSubmission",
    "nias-o:workflowSubject": {
      "@id": "nias:projects/{project-id}"
    }
  }
}
```

The report content node carries `nias-o:PddSectionAReport` content and links to
the project design fields generated by `PddSectionAUiShape`.

## PDD Section B Submission

Document schema:
`https://nova.org.za/novaimpactaccountingstandard/document-schema/PDDxB-9.0.0`

Expected SHACL coverage:

- `nias-o:DocumentShape`
- `nias-o:PddSectionBReportShape`
- `nias-o:ImpactRequirementShape`
- `nias-o:ImpactClaimShape`

Additional requirements:

- At least one declared impact.
- Monitored impact branch follows monitored impact requirements.
- Unmonitored impact branch follows fixed ex-ante requirements.
- SPARQL branch rule remains service-enforced until Fluree support is verified.

## PDD Section C Submission

Document schema:
`https://nova.org.za/novaimpactaccountingstandard/document-schema/PDDxC-4.0.0`

Expected SHACL coverage:

- `nias-o:DocumentShape`
- `nias-o:PddSectionCShape`
- `nias-o:PddSectionCReportShape`

Required content:

- stakeholder engagement modalities;
- stakeholder comment summary;
- stakeholder comment consideration;
- common document and workflow submission envelope.

## PDD Validation Review Submission

Document schema:
`https://nova.org.za/novaimpactaccountingstandard/document-schema/GenericDocumentReview-5.0.0`

Expected SHACL coverage:

- `nias-o:DocumentShape`
- `nias-o:GenericDocumentReviewShape`
- `nias-o:DocumentFieldReviewShape`

The generated PDD review forms now include `nias-o:isReviewOf`. This property is
required by the PDD-CIR gate.

```json
{
  "@context": "dataRequirements/fluree/context.jsonld",
  "@id": "nias:reviews/{project-slug}/{section}/review/{submission-id}",
  "@type": ["data:Document", "nias-o:GenericDocumentReview"],
  "nias-o:resourceIpfsUri": {
    "@type": "xsd:anyURI",
    "@value": "ipfs://..."
  },
  "nias-o:documentSchema": {
    "@id": "nias:document-schema/GenericDocumentReview-5.0.0"
  },
  "nias-o:isReviewOf": {
    "@id": "nias:documents/{project-slug}/{section}/{source-submission-id}"
  },
  "nias-o:fieldReview": [
    {
      "@type": "nias-o:DocumentFieldReview",
      "nias-o:reviewTarget": {
        "@type": "nias-o:ReviewTarget",
        "nias-o:reviewedArtifact": {
          "@id": "nias:documents/{project-slug}/{section}/{source-submission-id}"
        },
        "nias-o:reviewedAnchor": {
          "@id": "nias:documents/{project-slug}/{section}/{source-submission-id}/anchors/{anchor-id}"
        }
      },
      "nias-o:fieldTitle": "Project title",
      "nias-o:fieldPrompt": "Provide the project title.",
      "nias-o:originalResponse": "Original field value",
      "nias-o:reviewerDecision": {
        "@id": "nias-cs:review-approve"
      },
      "nias-o:reviewerFeedback": "Reviewer feedback."
    }
  ],
  "nias-o:finalReviewDecision": {
    "@id": "nias-cs:review-approve"
  }
}
```

Section-specific review steps:

| Section | Workflow step label | Reviewed document schema |
| --- | --- | --- |
| PDD-A | Validate PDD section A | `nias:document-schema/PDDxA-1.0.0` |
| PDD-B | Validate PDD section B | `nias:document-schema/PDDxB-9.0.0` |
| PDD-C | Validate PDD section C | `nias:document-schema/PDDxC-4.0.0` |

## PDD Certificate Issuance Request

Document schema:
`https://nova.org.za/novaimpactaccountingstandard/document-schema/PDDCIR-3.0.0`

Expected SHACL coverage:

- `nias-o:DocumentShape`
- `nias-o:PddCertificateIssuanceRequestShape`
- `nias-o:DocumentReferenceShape`

Gate prerequisite:

- The approved review set from
  `dataRequirements/shape2flutter/pdd-workflow-gate.md` must be resolved before
  creating this transaction.

```json
{
  "@context": "dataRequirements/fluree/context.jsonld",
  "@id": "nias:requests/{project-slug}/pdd-cir/{submission-id}",
  "@type": ["data:Document", "nias-o:PddCertificateIssuanceRequest"],
  "nias-o:documentSchema": {
    "@id": "nias:document-schema/PDDCIR-3.0.0"
  },
  "nias-o:pddSectionAValidationReview": {
    "@type": "nias-o:DocumentReference",
    "nias-o:documentMessageId": "0.0.1001-2026-01-04T00:00:00Z",
    "nias-o:resourceIpfsUri": {
      "@type": "xsd:anyURI",
      "@value": "ipfs://..."
    }
  },
  "nias-o:pddSectionBValidationReview": {
    "@type": "nias-o:DocumentReference",
    "nias-o:documentMessageId": "0.0.1001-2026-01-05T00:00:00Z",
    "nias-o:resourceIpfsUri": {
      "@type": "xsd:anyURI",
      "@value": "ipfs://..."
    }
  },
  "nias-o:pddSectionCValidationReview": {
    "@type": "nias-o:DocumentReference",
    "nias-o:documentMessageId": "0.0.1001-2026-01-06T00:00:00Z",
    "nias-o:resourceIpfsUri": {
      "@type": "xsd:anyURI",
      "@value": "ipfs://..."
    }
  },
  "nias-o:requestedIssuanceAccountId": "0.0.2002"
}
```

The workflow shell must not allow free-text review references to bypass the
gate. It should populate the three `DocumentReference` values from approved
lookup results only.
